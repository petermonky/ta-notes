Expressions

An expression is any valid code that evaluates to something.

A statement is an assertion that performs an action.


					// Expressions
					2
					true
					2 * 2

					// Statements
					2;
					true;
					2 * 2;

Expressions

Operations have precedence, abiding to given rules.


					// Start
					1 - 5 / 2 * 4 + 3;

					// Implicitly grouped and evaluated
					(1 - ((5 / 2) * 4)) + 3;

Naming and the Environment

Constant declarations create associations between names and values.


					// Constant declared
					const x = 2;

					// Name referenced
					x * x;

					// Value retrieved
					2 * 2;

					// Evaluated
					4;

Naming and the Environment

Names can be referred to from within constant declarations.


					// Constants declared
					const pi = 3.14;
					const radius = 2;

					// Names referenced
					const area = pi * radius * radius;

					// Values retrieved
					const area = 3.14 * 2 * 2;

					// Evaluated
					const area = 12.56;

Evaluating Operator Combinations

The evaluation rule is recursive in nature.

  1. Evaluate the operands

  2. Apply the operator action to the operands

To evaluate the whole, first evaluate its parts.

Evaluating Operator Combinations


					// To evaluate the whole...
					1 / 2 + 3 * 4;

					// ...first evaluate its parts.
					// Step 1
					0.5 + 3 * 4;

					// Step 2
					0.5 + 12;

					// Evaluated
					12.5;

Compound Functions

Function declarations are powerful in that they abstract away an operation to a name.

\[\texttt{function } name\texttt{(}parameters\texttt{)} \texttt{ \{ return } expression\texttt{; \}}\]

Like a constant, the function name may be referenced to call its operation.

Compound Functions

During function application, a declared function is called with necessary arguments, from which evaluation takes place.

\[function\texttt{-}expression\texttt{(}argument\texttt{-}expressions\texttt{)}\]
Yet another demonstration of the recursive nature of the evaluation rule.

Compound Functions


					// Function declared
					function square(x) {
						return x * x;
					}

					// Function called
					square(1 + 1);

					// Argument evaluated
					square(2);

					// Variable subsituted
					2 * 2;

					// Evaluated
					4;

Conditional Expressions and Predicates

Conditional expressions provide a means of changing control flow.

\[predicate\texttt{ ? }consequent\texttt{-}expression\texttt{ : }alternative\texttt{-}expression\]

The outcome expressions are evaluated conditionally after the conditional expression is evaluated.

Conditional Expressions and Predicates


					// Start
					0 === 0 ? 1 + 2 : 3 + 4;

					// Predicate expression evaluated
					true ? 1 + 2 : 3 + 4;

					// Conditional expression evaluated to consequent expression
					1 + 2;

					// Evaluated
					3;

The alternative expression is never evaluated.

Conditional Expressions and Predicates

Each operator combination has its own evaluation rule.

Conditional Expressions and Predicates

Multiple conditional expressions can be chained to check for multiple conditions.

\[ \begin{align*} &p_1\texttt{ ? }e_1\texttt{ : } \\ &p_2\texttt{ ? }e_2\texttt{ : } \\ &\vdots \\ &p_n\texttt{ ? }e_n\texttt{ : } \\ &final\texttt{-}alternative\texttt{-}expression \end{align*} \]

Conditional Expressions and Predicates


					// Function declared
					function add_one(x) {
						return x === 1 ? 2 :    // clause 1
						       x === 2 ? 3 :    // clause 2
							     x === 3 ? 4 :  // clause 3
							     0;
					}

					// Function called
					add_one(2);

					// Variable substituted
					2 === 1 ? 2 :
					2 === 2 ? 3 :
					2 === 3 ? 4 :
					0;

					// Predicate expression evaluated
					false ? 2 :
					2 === 2 ? 3 :
					2 === 3 ? 4 :
					0;

					// Conditional expression evaluated to alternative expression
					2 === 2 ? 3 :
					2 === 3 ? 4 :
					0;

					// Predicate expression evaluated
					true ? 3 :
					2 === 3 ? 4 :
					0;

					// Conditional expression evaluated to consequent expression
					3;

Conditional Expressions and Predicates

  • \(expression_1\texttt{ && }expression_2\)
    Syntactic sugar for \(expression_1\texttt{ ? }expression_2\texttt{ : }false\)

  • \(expression_1\texttt{ || }expression_2\)
    Syntactic sugar for \(expression_1\texttt{ ? }true\texttt{ : }expression_2\)

  • \(\texttt{!}expression\)
    Negates the expression
